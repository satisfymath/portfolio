# Riesgos y seguridad

- **Riesgo: infeasibilidad silenciosa** → se fuerza diagnóstico por slacks cuando procede.
- **Riesgo: fijación errónea** → validación de perfiles extraídos; fallback a iteración por componente.
- **Riesgo: explosión de dimensión** → soporte TSA (agregación + reconstrucción) y fijación por etapas.
- **Confidencialidad** → el repo público solo describe patrones; no incluye MPS ni inputs/outputs reales.
